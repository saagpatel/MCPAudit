# MCPAudit 1.5 Release Decision

Decision: ship `1.5.0` as adoption hardening, not beta.

## Evidence

- Local adoption examples now have smoke coverage that checks config-only
  defaults, report artifact upload behavior, and docs links.
- Config-health evidence includes redacted fixtures for project/global
  shadowing, remote credential headers, and shell-wrapped remote URL arguments.
- Prompt/resource scoring evidence includes issue tracker, browser automation,
  and resource-injection metadata cases in addition to the existing calibration
  corpus.
- The expanded corpus keeps benign memory/glossary cases quiet while surfacing
  high-risk prompt/resource metadata through `non_tool_risk`.

## Boundary

Do not change `risk_score.composite` in `1.5.0`.

The current evidence supports better documentation and validation of
`non_tool_risk`, but it does not prove that prompt/resource findings should be
merged into the stable tool-centered score. Keep any future combined score
additive and explicitly named.

## Next Release Bar

Move toward beta only after at least two external redacted reports show that the
current output contract is stable enough for downstream consumers. Until then,
prefer small adoption-hardening releases over scoring semantics changes.
